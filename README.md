# uncodixfy

A repo-ready markdown ruleset for avoiding generic AI-generated design while preserving broad creative freedom.

## Files

- `styles.md` — the main design guidance document
- `.gitignore` — basic ignore rules

## Purpose

This version of Uncodixify is intentionally broader and less prescriptive. It works as:

- an anti-pattern list
- a high-level taste filter
- a refinement pass against generic AI defaults

It is **not** meant to force one visual style, one layout system, or one repeated palette across projects.

## Suggested use

Use `styles.md` as a design refinement guide for:

- UI and frontend work
- websites and apps
- slides and presentations
- PDFs and documents
- other design-sensitive outputs
